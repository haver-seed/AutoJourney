# AutoJourney

多城市火车票智能规划工具。输入出发站与多个目的地，自动查询 12306 真实车次数据，通过智能算法选出性价比最优的车票方案。

## 功能

- **12306 实时查询** — 接入 12306 官方 API，获取真实车次、票价、余票信息
- **智能选票算法** — 综合价格、时间舒适度、跨日比价三维评分，自动推荐最优车次
  - 夜间长途车优先（节省住宿费）
  - 早间到达优先（最大化游玩时间）
  - 自动比较目标日期前后一天的价格
- **多城市串联规划** — 支持多个目的地，自动计算每段城际交通的出发日期
- **站点搜索** — 基于 12306 数据库的 5000+ 站点模糊搜索与自动补全
- **选票理由** — 每张推荐车票附带中文说明，解释为什么选择这趟车
- **实时进度** — SSE 流式推送查询进度

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Element Plus + Vite |
| 后端 | Python + FastAPI |
| 数据源 | 12306 官方 API + mcp_12306 站点库 |
| 站点解析 | mcp_12306（5000+ 站点模糊匹配） |
| 票价解码 | 12306 yp_info 字段解析（10字符分组：座位码 + 价格 + 编码） |

## 使用方法

### 第一步：填写出发信息

在页面顶部的表单中填写：

1. **出发站点** — 输入站名（如"北京南"），从下拉列表中选择。也可以点击下方热门站点标签快速填入。
2. **出发日期** — 选择出发日期，默认为 7 天后。

### 第二步：添加目的地

3. **目的地** — 输入城市名（如"西安"），点击"添加"或按回车。可以点击热门城市标签快速添加。
4. **调整停留天数** — 每个已添加的目的地右侧有停留天数输入框，默认 2 天，可自由修改（1-30 天）。
5. **调整顺序** — 每个城市右侧有上下箭头按钮，可以拖动调整访问顺序。

示例：添加"西安"（2天）、"成都"（3天）、"重庆"（2天）

### 第三步：查询车票

6. 点击 **"查询车票"** 按钮，系统会逐段查询 12306 真实车次。
7. 查询过程中会显示实时进度条。

### 查看结果

查询完成后，页面显示：

- **汇总栏** — 总行程天数、总票价、共几段
- **车票方案列表** — 每段行程的详细信息：
  - 车次号（如 G655）
  - 出发/到达时间和历时
  - 座位类型和票价（二等座、一等座、商务座）
  - 选票理由（如"价格最低"、"夜间出发，节省一晚住宿费"、"08:30 到达，时间充裕可直接游玩"）
  - 停留天数标签

### 示例路线

| 路线 | 天数 | 说明 |
|------|------|------|
| 北京南 → 西安 → 成都 → 重庆 | 7天 | 经典西北-西南串联 |
| 上海虹桥 → 杭州 → 南京 → 苏州 | 5天 | 长三角精华游 |
| 广州南 → 桂林 → 长沙 → 武汉 | 7天 | 南北纵贯线 |

## 快速开始

### 环境要求

- Python 3.10+
- Node.js 18+

### 安装依赖

```bash
# 后端
cd backend
pip install -r requirements.txt

# 前端
cd frontend
npm install
```

### 启动服务

需要同时运行后端和前端，**打开两个终端窗口**：

**终端 1 — 启动后端（端口 8000）：**
```bash
cd backend
python -m uvicorn main:app --reload --port 8000
```

**终端 2 — 启动前端（端口 3000）：**
```bash
cd frontend
npm run dev
```

### 访问页面

浏览器打开 `http://localhost:3000`，看到 AutoJourney 页面即启动成功。

> 前端开发服务器会自动将 `/api` 请求代理到后端 8000 端口，无需额外配置。

## API 接口

### `GET /api/stations?q=北京&limit=15`

站点搜索，返回匹配的站点列表（名称、编码、所属城市）。

### `POST /api/tickets`

查询多段城际车票，SSE 流式返回。

**请求体：**
```json
{
  "departure": "北京南",
  "departure_date": "2026-05-15",
  "destinations": ["西安", "成都", "重庆"],
  "stay_days": [2, 3, 2]
}
```

**SSE 事件：**
- `progress` — 查询进度（`{ stage, percent, message }`）
- `result` — 最终结果（`{ segments, total_cost, total_days }`）
- `error` — 错误信息

### `GET /api/health`

健康检查，返回 `{"status": "ok"}`。

## 智能选票算法

每趟候选列车经过三维评分：

1. **价格分**（权重最高）— 归一化到 0-100，优先选二等座价格
2. **时间舒适度分** — 夜间长途 +30，早到 +20，晚到 -15，早发 +15
3. **日期偏离惩罚** — 非目标日期 +5 分惩罚

最终得分 = 价格分 + 时间分 + 日期惩罚，取最低分者为推荐车次。
